## Class Notes 
- What did we find in the TODO app?
    - Where is the script tag in my todo.html?
        - It is near the end. The opening tag is in line 271 and the closing one is in 432, with having 434 lines in total (Remember that script is the last part of the body).
    - How does javascript find html elements on the page?
        - JavaScript uses document.getElementById() to search the page for any element that has a matching id. It is like every HTML element having a nametag while JavaScript reads the nametag to find exactly what it is looking for. Once it finds the elemtn, it saves it in a variable so it can be able to use it later.
    - What makes the "add" button actually do something?
        - When you click it, it triggers the "submit" action. The code is listening for that action to happen. When it does happen, it reads whatever text you typed into the input box, stores it as a new todo object (with id, text, category, and completed/starred status both set to false), and it then pushes it into the todos list. Then it saves the updated array and redoes the list. 
    - Where does the code change the page (Add text, add items, etc...)?
        - This happens inside a function called renderTodos(). Every time something changes — adding, deleting, completing, or starring a task — the code wipes everything clean and makes it empty. Then it will loop through all the todos and build brand new HTML elements from scratch using document.createElement(), fills them with text and buttons, and drops them onto the page using appendChild(). So the page never truly "updates", it just gets  redrawn every time.
