Review questions to know:
- if you inspect a website and see display:flex, on a container with 5 items that are stacked vertically, what property is causing this: flex-direction: column — it changes the flex main axis to vertical so the items stack top-to-bottom (also possible: flex-direction: column-reverse or the shorthand flex-flow).
- fontawesome is for icons, not emojis; you can ask copilot to add them for me

- We learned how to create a form using code. This form can be added to any website to collect information
- action in a form: where to send the data (a URL or server endpoint)
- method: 
    - GET: the data in the URL, use for search/filtering (results are bookmarkable)
    - POST: data in the request body, use for login, checkout, or sensitive data
    - MDN: sending form data
- Password: If you use GET method, the information will appear on the search bar (soemthing that you dont want to happen with sensitive information such as your password)
    - the correct way to create a password field in a form is using POST (so that it does not show up in the search bar and will not be saved in your history data)
- Echo test form will return your form back to you after it is submitted
- Responsive design: websites that can adapt to different screen sizes
    - viewport meta tag is essential for responsive design
    - without it, mobile browsers will assume that your site is desktop only
    - < meta name = "viewport" content ="width=device-width, >
- Mobile first approach: 
    - design for mobile first (smallest screen)
    - add complexity for larger screens
    - results in faster mobile sites
    - forces you to prioritize content
