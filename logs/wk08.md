## TODO App Findings
1. Where is the < script > tag?
- The script tag is placed at the bottom of the HTML file, right before the closing </body> tag. This is useful because the page loads the HTML first, and then runs the JavaScript after the elements already exist. 
2. Find every addEventListener() + what does each one listen for?
- there are 4 in total:
    - One listens for the "change" event on each checkbox. This updates whether a todo is completed.
    - One listens for the "click" event on each Delete button. This removes that specific todo item.
    - One listens for the "submit" event on the todo form. This adds a new todo when the user submits the form.
    - One listens for the "click" event on the Clear Completed button. This removes all completed todos at once.
3. Where does the code change the DOM? (textContent, innerHTML, classList, appendChild, etc...)
- it changes in many places: 
    - It updates text using textContent (for the items-left counter, todo text, and Delete button label).
    - It clears and rebuilds the todo list using innerHTML = "" and then creating new elements.
    - It creates new HTML elements with document.createElement(...) for each todo row.
    - It changes CSS classes with className (for completed todos and button styling).
    - It inserts elements into the page using append(...) and appendChild(...).
    - It updates form/input UI values like checkbox.checked and resetting todoInput.value after adding a task.
