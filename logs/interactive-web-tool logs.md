## Iteration 1: Structure + First Interaction
- What HTML structure did you choose? What was your first piece of JavaScript? If AI generated it, what did you have to understand to make it work with your HTML? 
    - For my HTML sturcture I decided to use a simple one: header, add-card form, and display area. I used semantic tags like < header >  and < button > instead of divs everywhere. The card is built with 2 nested divs (.card-scene) and (.card-inner) with teo (.card-face) divs inside it, one for the front and one for the back. This is what makes the 3D flip work. My first piece of JavaScript was the add card function, when you click the button, it reads the two input fields, pushes an obkect into the cards array and calls render() to update the entire page. Even though AI generated most of it, I had to understand how the cards array stores the objects, what .trim() does to the input values and why render() had to be called every time state changes instead of just updating one element directly everytime. 

## Iteration 2: Core Functionality 
- What was the hardest interaction to implement? What JavaScript concepts did you use (variables, functions, arrays, conditionals, DOM methods)? What bugs did you find and fix? 
    - The hardest interaction to put was the card flip I believe. It is not just JavaScript that I had to use, I needed CSS perspective on the outer container, transform-style: preserve-3d on the inner container, and backface-visibility: hidden on each face so the wrong side isn't visible. Getting all 3 of those things to work together was hard to do but I eventually got it done. 
    - The JavaScript concepts that I used: variable(cards, currentIndex, isFlipped) to track state, functions to seperate logic (addCard, goNext, flipCard), arrays to store and loop through all the cards, conditionals to check if there were any empty inputs and DOM methods like classList.toggle, textContent, and innerHTML to fully update the whole page. 
    - Bugs that I had found: if someone deleted the last card in the deck the entire index broke and broke the display. I also found that typing only spaces into it would pass the empty card so I had to add .trim() to fix that. 

## Iteration 3: Polish + Test
- What feedback did you get? What edge cases did you handle? What would you add with more time? 
    - I had my roomate test it out and the first thing they said was that they did not realize you could click the card to flip it, so I added the "click to flip" text on the card so any user knows. They also said that when they scrolled down you can see all the answers in the deck list, which ruins the point of studying if you can see thr answers. So, I added blur on the answers by default with a reveal toggle. 
    - Edge cases I handled: empty filter results (if you go to "known" with no cards there, it shows a message instead of it all just crashing), deleting the card you're currently on, and adding a card to the "still learning" filter. 
    - I would add localStorage with more time so my cards save when you close the tab. I would also add the chance to edit a card after you created it, since right now you have to delete and re-add it if you have a typo. I could have also added a quiz section, where it automatically creates a quiz for yout to take with all the cards you created.
